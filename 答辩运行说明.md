# Lyrics Map 答辩运行说明

## 1. 换电脑运行

1. 把整个 `web-prototype` 文件夹复制到另一台电脑。
2. 确认电脑已安装 Node.js 18 或更新版本。
   - 下载地址：https://nodejs.org/
   - 安装后打开命令行输入 `node -v`，能看到版本号即可。
3. 双击 `start-web.bat`。
4. 浏览器打开：
   - 本机访问：http://localhost:4174
   - 同一 Wi-Fi / 热点下的其他设备访问：启动窗口里打印的 `http://本机IP:4174`

启动窗口不要关闭，关闭后网站也会停止。

## 2. 答辩现场最稳方案

优先使用自己的电脑本机演示：

- 先双击 `start-web.bat`
- 打开 http://localhost:4174
- 提前点开 `目的地攻略`、`歌曲收录`、`行程规划表`、`AI旅行歌单`
- 保持启动窗口、浏览器页面都不要关闭

建议准备手机热点。如果教室网络不稳定，电脑连手机热点后重新启动 `start-web.bat`。

## 3. 让其他电脑或手机访问

1. 演示电脑和访问设备连同一个 Wi-Fi 或同一个手机热点。
2. 在演示电脑启动 `start-web.bat`。
3. 看启动窗口里的地址，例如 `http://192.168.1.8:4174`。
4. 其他设备浏览器输入这个地址。
5. 如果打不开，在 Windows 弹出的防火墙提示里允许 Node.js 访问专用网络。

## 4. 联网要求

页面主体可以由本地服务打开，但以下能力需要网络：

- 高德地图底图和地图标记渲染
- 外部来源链接，如 Google Maps、小红书、豆瓣、Agoda
- AI 旅行歌单接口

如果网络不稳定，网站页面仍能打开；地图区域可能显示加载失败提示。答辩前建议准备截图或录屏作为备用。

## 5. AI 页面说明

AI 页面需要 API Key 才能真正调用接口。没有 Key 时，页面会把提示词展示出来，可手动复制到 DeepSeek / 豆包官网继续演示。

不要把真实 AI API Key 公开发给别人或上传到公开仓库。

## 6. 公网访问方案

如果需要老师不在同一 Wi-Fi 下也能访问，有两种方式：

- 临时公网演示：使用 ngrok 或 cloudflared，把本地 `4174` 端口映射成公网链接。
- 长期部署：把 `web-prototype` 部署到云服务器或 Render / Railway 等 Node 托管平台，启动命令为 `node server.mjs`。

GitHub Pages 这类纯静态托管能展示大部分页面，但 `/config.js`、AI API 代理和根路径配置可能需要额外调整；答辩优先推荐本机运行或 Node 托管。
